Building and Running JDiff

JDiff comes with a set of Windows batch files to compile JDiff and run the unit tests, These are in the directory "build". Make sure you are in the build directory and have set the environment variable JAVA_HOME to the root of your current JDK installation. and then execute one of the following scripts. Do inspect the output carefully for failures, since batch files do not stop at failure, by default.

  • To build JDiff from source, type: .\makejdiff
  • To build JDiff tests from source, type: .\maketests
  • To run JDiff tests, type: .\runtests
    Note that a warning about "a method no longer being deprecated" is an expected part of the output when running this batch file.

The following scripts assume you have source code for each JDK locally and have edited the scripts to indicate where the source code is located.

  • To create XML for the JDK1.2 API, type: .\jdk12toXML
  • To create XML for the JDK1.3 API, type: .\jdk13toXML
  • To generate a report showing the differences between the two JDK versions, type: .\diffJDK

Running prepare_release will clean all the directories and prepare the JAR file and ZIP file for shipping JDiff. Be warned that it will remove all generated XML files.
